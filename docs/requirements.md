# 需求文档

## 一、用户模块
### 1.1 用户注册
目前要求支持邮箱注册，使用QQ SMTP服务发送邮件，验证码，验证码分注册，登录等类型，不桶类型验证码无法混用
### 1.2 用户登录
支持多模式登录，用户可以输入账号/用户名/邮箱，密码登录。也支持使用邮箱发送验证码登录。登录后使用JWT校验身份，先只做USER,VIP,ADMIN三种角色
### 1.3 用户修改基本身份信息
用户名和账号无法修改
可以修改昵称，头像，密码等，邮箱修改时需要验证修改前后的邮箱，都通过后再修改

### 1.4 用户注销
用户注销时，需要先验证用户身份，发送验证码信息，用户确定后，有7天冷静期，冷静期期间，用户如果登录，则不能注销，7天后，永久删除用户信息（逻辑删除）
